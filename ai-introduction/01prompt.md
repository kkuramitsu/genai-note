# 生成AIを使ってみる


### ゴール

- クラスの生成AI利用実態を把握し、自分の使い方を客観視する
- **プロンプト**の書き方次第で回答が大きく変わることを体感する
- **ハルシネーション**という生成AI特有の現象を理解し、自分で確認する


## 1. 代表的な生成AI



チャット型の生成AIはいくつかあります。
いずれも無料プランで基本的な機能を試せます。

| ツール | 開発元 | 特徴 |
|--------|--------|------|
| **ChatGPT** | OpenAI | 生成AIの代名詞。ユーザ数が最も多く、情報・事例が豊富 |
| **Claude** | Anthropic | 日本語の文章品質とプログラミング能力の高さに定評がある |
| **Gemini** | Google | 無料で使える利用枠が広く、Googleサービスと連携しやすい |
| **Grok** | xAI | X（旧Twitter）の最新情報を学習しているのが特徴 |

### ChatGPT (ちゃっぴー)

開発元は**OpenAI**（アメリカ、2015年設立）。2022年11月の公開直後に世界中で話題となり、「生成AI」という言葉を一般に広めた立役者です。Microsoft が巨額出資しており、Windows や Office の Copilot にも同系統の技術が使われています。

![](../images/ai/chapgpt4-fs8.png)
*おなじみのちゃっぴー(少し前のバージョン)*

https://chatgpt.com

ユーザー数・関連情報・事例がいずれも最多で、困ったときにWeb検索で解決策が見つかりやすいのが強みです。画像生成（DALL·E）・音声会話・データ分析など機能も総合的にそろっており、「迷ったらまずこれ」という位置づけです。

### Gemini (ジェミニ)

開発元は**Google DeepMind**（Google の AI 研究部門）。2023年に Bard として公開され、2024年に Gemini へ改称されました。検索エンジン世界最大手 Google の技術と検索インフラが背景にあります。

![](../images/ai/gemini2-fs8.png)
*Gemini(少し前のバージョン)*

https://gemini.google.com/

Gmail・Googleドキュメント・YouTube など Google サービスとの連携が自然で、大学生が日常的に使うツールと相性が良いのが特徴です。無料で使える枠が比較的広く、画像・音声・動画も同じモデルで扱える**マルチモーダル**設計が売りです。Android 端末では標準アシスタントとしても使えます。

### Grok (グロック)

開発元は**xAI**（Elon Musk が2023年に設立）。X（旧 Twitter）と同じ運営陣で、X のリアルタイム投稿データを学習・参照に使えるのが最大の特徴です。

![](../images/ai/grok3-fs8.png)
*Grok(少し前のバージョン)*

https://grok.com/

「今この瞬間、SNSで話題になっていること」を尋ねるのが得意で、ニュース・スポーツ・トレンドの即時把握に向きます。他の生成AIに比べて**回答の規制がゆるめ**に設計されており、ジョークや皮肉も返す反面、情報の偏りや過激さには注意が必要です。利用には X のプレミアムプラン加入が前提となる場面が多いです。

### Claude (クロード)

開発元は**Anthropic**（アメリカ、2021年設立）。創業者の Dario Amodei・Daniela Amodei 兄妹は元 OpenAI の研究責任者で、**AIの安全性（AI Safety）**を前面に掲げて会社を立ち上げました。Google と Amazon が大規模に出資しています。

![](../images/ai/claude3-fs8.png)
*Claude(少し前のバージョン)*

https://claude.ai/

日本語の文章が自然で、長文の要約・推敲・翻訳に強いとの評価が定着しています。**プログラミング能力も高く、コード支援ツール Claude Code の基盤にもなっています。**過剰に迎合せず、慎重に答える性格に調整されているため、レポート執筆や議論の壁打ち相手として使いやすいです。

> [!NOTE]
> #### どれを使えばいいか？
>
> この授業では、まず無料プランで色々試してみて特徴を把握してみることが大切です。
>
> 各社が競うように新モデル・新機能をリリースしており、情報はすぐ古くなります。
> ユーザーレビューも参考になりますが、求める能力は人それぞれ異なります。
>
> 本格的に活用するときは、自分の目的にあった生成AIを制限を気にすることなく使えるように課金も検討していいと思います。

## 2. プロンプトが全て

**プロンプト（prompt）**とは、生成AIへの指示と、その指示を補足するコンテキスト（文脈）のことです。

同じ質問でも、プロンプトの書き方次第で回答は劇的に変わります。

![](../images/ai/prompt-fs8.png)

### 役割を与える

まず、プロンプトの書き方次第で大きく結果が変わる例をみましょう。

次の2つのプロンプトを実際に試してください。

#### プロンプト A

![AIの社会課題を教えてください](../images/callout/prompt.svg)

#### プロンプト B

![あなたは高校の先生です。AIの社会課題を教えてください](../images/callout/prompt.svg)


回答はどう変わりましたか？

生成AIは「あなたは〇〇です」という形で**役割（ロール）を与える**と、その立場に即した回答をしてくれます。

![](../images/ai/prompt_role-fs8.png)

| 役割の例 | 用途 |
|---------|------|
| 「あなたは内科医です。患者が咳と微熱を訴えています」 | 症状の初期情報収集 |
| 「あなたは弁護士です。この契約書のリスクを教えてください」 | 法的リスクの確認 |
| 「あなたは編集者です。この文章をわかりやすく改善してください」 | 文章の推敲 |
| 「あなたは批評家です。この案の弱点を指摘してください」 | アイデアの検証 |


> [!WARNING]
> **注意**：生成AIは医師・弁護士ではありません。役割を与えた回答はあくまで参考情報です。重要な判断は専門家に確認してください。


### プロンプトにまつわる話

生成AIのユーザーたちが試行錯誤する中で、「こうすると回答が変わる」という発見がいくつか共有されています。科学的に厳密に検証されたものもあれば、そうでないものもありますが、実際に試してみる価値はあります。

| 試み | 内容 |
|------|------|
| **例を見せること** | 具体的に例を見せると回答の正確さが上がります |
| **ステップで考えさせる** | 「順を追って考えてください（Let's think step by step）」と加えると、推論を伴う問いで回答の質が上がる傾向がある |
| **励ます** | 「あなたは優秀です」「頑張ってください」など励ましの言葉を添えると回答が改善されることがあるとの報告がある |
| **繰り返す** | 重要な指示を2回書くと、無視されにくくなるという経験談がある |
| **英語で書く** | 同じ質問を英語で送ると、学習データの量の違いからより詳細な回答が返ることがある |

いずれも「必ず効く」とは言い切れません。モデルや質問の内容によって結果は変わります。ただ、**なぜ効くのか（あるいは効かないのか）を考えること**が、生成AIの仕組みへの理解を深めるきっかけになります。

## 3. ペルソナを使った活用

### ペルソナとは

**ペルソナ（Persona）**とは、架空の人物像を具体的に設定したものです。名前・年齢・専攻・性格・生活習慣・口癖など、その人らしさを決める情報を組み合わせて作ります。

![](../images/ai/persona-fs8.png)
*ペルソナの例(講義スライドより)*

もともとはデザインや製品開発の分野で「ターゲットユーザーを具体的にイメージするための手法」として使われてきました。生成AIにペルソナを与えると、その人物として会話してくれます。

**ペルソナは個人情報の代わりになります。** 自分自身の名前・学籍番号・住所を入力する必要はありません。「自分に似た架空の人物」を設定することで、プライバシーを守りながら生成AIを活用できます。

> [!WARNING]
> ### 注意：個人情報を入力しない
>
> プロンプトに入力したテキストはサーバーに送られ、場合によってはAIの学習データに使われることがあります。**氏名・住所・学籍番号・パスワードなど、個人を特定できる情報は絶対に入力しないでください。**


### ペルソナ作成のコツ

| コツ | 説明 |
|------|------|
| **属性を具体的に書く** | 「大学生」より「理工学部2年、一人暮らし、週3バイト」のほうがAIの反応が豊かになる |
| **矛盾をなくす** | 「内向的だが学園祭実行委員長」のように矛盾があると、AIが一貫した回答をしにくくなる |
| **「らしさ」を一言加える** | 口癖・趣味・価値観など、その人だけの特徴を一つ入れると個性が出る |
| **実在の個人情報は使わない** | 自分や他人の本名・連絡先は書かない。「架空の人物」として設定する |

ペルソナの精度が高いほど、生成AIの回答は「その人っぽく」なります。

### 演習：ペルソナを作って会話する

**ステップ1**：ペルソナを設定する

自分に似た架空の人物を自由に設定してください。実際の個人情報は一切使わなくて構いません。

```
例：
「名前は A さん（仮）。文学部3年生。学園祭の実行委員を2年間務め、
 チームをまとめる役を担うことが多い。議論をまとめるのが得意だと
 周りから言われるが、本人はそれを強みとは思っていない。
 趣味は読書と映画鑑賞。口癖は「とりあえずやってみよう」。」
```

> [!WARNING]
> 生成AIの会話は、それまでのチャットを全て覚えています。
> 新しいペルソナを与えるときは、新しいチャットを始めるようにしてください。


**ステップ2**：ペルソナを渡して会話する

次のようなプロンプトで、ペルソナと会話してみてください。

```
「上記のペルソナの人物になりきって、私の質問に答えてください。
 まず自己紹介してください。」
```

**ステップ3**：「本人っぽいか」を評価する

AIの回答を読んで、ペルソナとして設定した人物らしいかどうか確認してください。

- 設定した性格や口癖は反映されているか
- 「そのキャラクターとしては言わなそう」な回答はないか
- ペルソナの情報が少ないと、AIの回答はどう変わるか

> **考えてみよう**：ペルソナの設定が薄いほど、AIは「ありきたりな人物」として回答します。
どんな情報を加えると「その人らしさ」が増しましたか？

> [!NOTE]
> 面白いペルソナが作れたらペルソナと会話例を Google ドキュメントに追記してみましょう。


## 4. ハルシネーション

プロンプトを工夫することで生成AIは非常に役立ちます。しかし、重大な特性があります。

**ハルシネーション（Hallucination：幻覚）**とは、AIが**自信を持って誤った情報を出力する**現象です。

生成AIは「知らない」と言うより、それらしい回答を作り上げてしまう傾向があります。

![](../images/ai/snowman-fs8.png)
*どこが間違っているかわかりますか？*

> [!WARNING]
> 上の例は、ChatGPT3.5(2023年頃)の例です。
> もちろん、最新の生成AIはSnowManのメンバーを間違えることはありません。


### なぜハルシネーションが起きるか

生成AIは「次にくる言葉として確率的に自然なもの」を生成します。「正しいかどうか」ではなく、「それらしい文章として続くかどうか」を優先するため、間違っていても自信に満ちた文章を作ってしまいます。

> [!NOTE]
> **生成AIが得意なこと：**
> * 既存の知識を整理・説明する
> * 文章を書く・翻訳する
> * コードを生成する


> [!NOTE]
> **ハルシネーションが起きやすい場面：**
> * 最新の情報（学習データの締め切り以降の出来事）
> * マイナーな固有名詞（人名・地名・作品名）
> * 正確な数値・日付・統計
> * 引用・参考文献の出典


### ハルシネーションは原理的になくせない

ハルシネーションは技術の未熟さから来る一時的な問題ではなく、現在の生成AIの仕組みに根ざした本質的な限界です。

生成AIは膨大なテキストから「言葉のパターン」を学習しています。内部には「事実を格納するデータベース」のようなものは存在せず、知識と推論と創作が区別されないまま混在しています。そのため、モデル自身は「自分が確実に知っていること」と「それらしく生成していること」を区別できません。

また、モデルの学習目標は「次の単語を正確に予測する」ことであり、「事実を正確に記憶する」ことではありません。様々な改善手法が研究されていますが、この根本的な生成の仕組みが変わらない以上、ハルシネーションをゼロにはできません。

> **重要な前提**：生成AIを使う人は全員、「この回答は間違っているかもしれない」という前提を持ち続ける必要があります。これは特定のサービスが優れていないからではなく、現在の技術の構造上避けられないことです。

### ハルシネーションが引き起こす社会問題

ハルシネーションは単なる「AIの間違い」ではなく、現実の社会に被害を与えうる問題です。

| リスク領域 | 具体例 | 影響の深刻度 |
|-----------|--------|-------------|
| 法律・医療 | 存在しない判例・診断基準の引用 | 高（生命・権利に直結） |
| ジャーナリズム | 誤情報の記事化・拡散 | 高（社会的影響が広い） |
| 学術・教育 | 架空論文の引用、誤統計 | 中（信頼性の毀損） |
| 個人の名誉 | 誤った人物情報の生成・流通 | 中〜高（法的リスク） |
| 日常的な調査 | 店舗情報・製品仕様の誤り | 低〜中（実害は限定的） |


**法律・医療分野での誤情報**

弁護士がAIの生成した判例を裁判書類に引用したところ、存在しない判例だったという事例がアメリカで複数報告されています（2023年）。医療・法律・会計など、専門的な事実確認が必要な分野でハルシネーションが起きると、誤った判断につながる危険があります。

**フェイク情報の拡散**

ハルシネーションで生成された誤情報が、そのままSNSやニュースサイトに転載されることがあります。「AIが言っていた」という言葉が権威のように扱われ、誤情報の信頼性を高めてしまう逆効果も指摘されています。

**学術・教育への影響**

存在しない論文の引用、誤った統計データの引用がレポートや論文に混入するリスクがあります。生成AIを使って調査・執筆を行う場合、出典の一次確認は必須です。

**名誉毀損・風評被害**

実在する人物について誤った経歴・発言・犯罪歴などをもっともらしく生成する事例があります。本人が知らないうちにインターネット上に流通するケースもあり、法的問題に発展する可能性があります。

> [!WARNING]
> ハルシネーションは現在も完全には解決されていない技術的課題です。「便利なツール」として活用するためには、その限界を正確に理解することが前提になります。


### ハルシネーションを防ぐには

| 対策 | 内容 |
|------|------|
| **明確なプロンプト** | 曖昧な質問は余計な推測を生む。具体的に聞く |
| **ファクトチェック** | 重要な情報は必ず一次情報で確認する |
| **最新モデルを使う** | 新しいモデルほどハルシネーションは減少傾向 |
| **「知らない場合は教えて」** | プロンプトに「わからない場合は正直に言ってください」と加える |


### 演習：ハルシネーションを発生させる

生成AIの限界を知るには、自分でハルシネーションを「起こしてみる」のが効果的です。

> [!NOTE]
> **やってみよう**
> 1. 自分がよく知っている分野（好きなバンド・アニメ・スポーツ選手など）について
>    詳しい情報を生成AIに聞く
> 2. 返ってきた回答に間違いがないか確認する
> 3. どこが間違っていたかを記録する


**仮説を立ててから試してみましょう**：
- 「〇〇の分野は間違えそう」という予想は当たりましたか？
- どんな種類の情報が間違いやすかったですか？

> 「あえて自分のよく知っている情報を調べると、ハルシネーションを見つけやすい。限界を体験すれば、生成AIに過度に依存しなくなる。」（受講生の感想より）


## まとめ

- **プロンプトが全て**：役割・文脈・具体性を加えるだけで回答の質は大きく変わる
- **ペルソナで個人情報を守る**：架空の人物像を使えば、プライバシーを守りながら深く活用できる
- **ハルシネーションは必ず起きる**：特に固有名詞・数値・引用に注意
- **個人情報は入力しない**：この習慣を今日から徹底する

- b) 生成AIがその立場に合った視点と語調で回答するようになる

- c) 生成AIを無料で使えるようになる

**Q5．** 本日の演習で試したプロンプトの工夫として、回答の質が上がる可能性が最も高いものはどれか。

- a) プロンプトを短くする
- b) 役割・文脈・欲しい形式を具体的に書く
- c) 同じ質問を何度も送る
- d) 絵文字を多く使う

<details>
<summary>答え</summary>

- Q1：**b**　生成AIは「確率的に自然な文章」を生成するため、誤情報でも自信満々に出力する。
- Q2：**b**　役割を与えると、その立場に沿った視点・語調で回答する。ハルシネーションや情報の鮮度は変わらない。
- Q3：**b**　ペルソナは架空の人物像であり、個人情報を入力せずに自分に近い文脈で活用できる。精度が100%になるわけではない。
